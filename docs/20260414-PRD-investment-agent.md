# Product Requirements Document
# AI Investment Agent — US Stock Momentum Trader

**Version:** 1.1
**Date:** 2026-04-14
**Owner:** Gary
**Status:** Approved

---

## 1. Problem Statement

Gary wants to actively learn US stock investing but lacks the time and expertise to screen stocks, interpret technical signals, and make timely decisions daily. Manual research is slow, inconsistent, and emotionally driven. The goal is a personal AI agent that does the analysis, executes paper trades autonomously, and produces a daily HTML dashboard — so Gary can learn by doing, with guardrails.

---

## 2. Product Goal

Build a fully autonomous AI agent that manages a $10,000 simulated portfolio, targeting a **15% return ($1,500 profit) within 30 days**, using momentum-based swing trading on US stocks. The agent runs on a schedule with no human confirmation required.

### Success Metrics

| ID | Metric | Target |
|---|---|---|
| G1 | Starting capital | Exactly $10,000 virtual cash |
| G2 | Autonomous operation | Runs every trading day without manual trigger |
| G3 | AI decision-maker | Claude is the sole decision-maker for all trades |
| G4 | Full audit trail | Every trade stored with timestamp, price, and Claude's reasoning |
| G5 | Risk enforcement | 4 hard rules enforced in code, not just in the prompt |
| G6 | Plain-English commentary | Daily summary printed and saved after every run |
| G7 | 30-day performance tracking | `history` command shows daily P&L and all trades |

---

## 3. Users

**Primary user:** Gary (solo user, personal use only)

## 5. Functional Requirements

### 5.1 Scheduling

- FR-01: Run 1 executes automatically at 6:00 AM PT every weekday (Monday-Friday)
- FR-02: Run 2 executes automatically at 6:30 AM PT every weekday
- FR-03: Agent does not run on weekends or US market holidays
- FR-04: Agent can also be triggered manually via CLI command

### 5.2 Run 1 — Morning Briefing (6:00 AM PT)

- FR-05: Scan watchlist of 15-20 high-liquidity US stocks
- FR-06: Score each stock using NASDAQ/S&P500 market direction + individual stock technicals (RSI, MACD, volume)
- FR-07: Check overall market direction: if NASDAQ or S&P500 is down >1% pre-market, flag as risk-off day
- FR-08: Generate a ranked list of top buy candidates and any positions to sell
- FR-09: Save the plan to a JSON file for Run 2 to consume
- FR-10: Print a plain-English briefing: market conditions, top picks, reasoning for each

### 5.3 Run 2 — Execution (6:30 AM PT)

- FR-11: Read the plan generated by Run 1
- FR-12: Execute all buy and sell orders at current yfinance market price
- FR-13: Store each trade in SQLite with: timestamp, ticker, action, shares, price, total cost, Claude's reasoning
- FR-14: Apply all 4 hardcoded risk rules before executing any order (see FR-20 to FR-23)
- FR-15: Print confirmation of each executed trade

### 5.4 Intraday Monitoring

- FR-16: Every 30 minutes during market hours, check open positions against stop-loss and profit targets
- FR-17: Auto-sell any position that hits -7% stop-loss
- FR-18: Auto-sell any position that hits +12% profit target
- FR-19: Log all auto-sells with reasoning: "stop-loss triggered" or "profit target hit"

### 5.5 Hardcoded Risk Rules (enforced in code, not prompt)

- FR-20: Max 25% of total portfolio value in any single position ($2,500 on a $10,000 base)
- FR-21: Stop-loss at -7% — auto-sell, no exceptions
- FR-22: Profit target at +12% — auto-sell to lock in gains
- FR-23: Maximum 3 open positions simultaneously — no new buys if 3 positions held

### 5.6 Portfolio Monitoring

- FR-24: Portfolio state persists in SQLite across sessions
- FR-25: On any run, agent knows: cash balance, open positions, avg cost, current P&L
- FR-26: `history` command prints all trades with Claude's reasoning per trade

### 5.7 HTML Dashboard

- FR-27: Dashboard HTML file is regenerated after every Run 2 and every end-of-day monitoring cycle
- FR-28: Dashboard shows: total portfolio value, cash, P&L%, progress bar toward 15% target
- FR-29: Dashboard shows open positions table: ticker, shares, avg cost, current price, unrealized P&L%
- FR-30: Dashboard shows trade history table: date, action, ticker, shares, price, reasoning
- FR-31: Dashboard shows a 30-day P&L line chart (daily portfolio value over time)
- FR-32: Dashboard is a single self-contained HTML file — no server needed, opens in browser

## 6. Non-Functional Requirements

| Category | Requirement |
|---|---|
| Response time | Run 1 completes in under 90 seconds; Run 2 in under 60 seconds |
| Availability | Runs locally on Gary's Mac; no server dependency |
| Data freshness | yfinance (15-min delayed); acceptable for swing trading |
| Privacy | No user data sent anywhere except Anthropic API (query text) |
| Persistence | All portfolio state and trade history in SQLite |
| Cost | Anthropic API cost < $5/month at normal daily usage |
| Dashboard | Self-contained HTML file, no dependencies, opens in any browser |

---

## 7. Out of Scope (v1.0)

- Voice interface
- Real money trading / brokerage integration
- Options, futures, crypto, or non-US markets
- Mobile app
- Push notifications or alerts outside the scheduled runs
- Multi-user support
- Backtesting historical strategies
- News sentiment analysis

---

## 8. Assumptions

- Gary's Mac is on and connected to the internet weekdays by 6:00 AM PT
- Gary uses a cron job or scheduler to trigger the agent automatically
- yfinance 15-minute delayed data is acceptable for swing trade decisions
- Gary understands this is paper trading; no real money is at risk in v1.0
- US market hours are 9:30 AM - 4:00 PM ET (6:30 AM - 1:00 PM PT)

---

## 9. Risks

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| yfinance API breaks (Yahoo changes endpoints) | Medium | High | Pin yfinance version; monitor for breakage |
| Mac asleep at 6:00 AM PT | Medium | High | Enable "wake for network access" in Mac energy settings |
| 15% target not achieved in 30 days | High | Low | Educational — learning the strategy is the real value |
| Agent takes excessive losses before stop-loss triggers | Low | Medium | FR-21 hardcoded stop-loss at -7% enforced in code |
